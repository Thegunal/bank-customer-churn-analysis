# Bank Customer Churn Analysis

## Overview

An end-to-end Data Analytics project focused on understanding customer churn patterns in a European bank using Python, MySQL, and Power BI.

The project aims to identify key churn drivers, analyze customer behavior, and provide actionable business recommendations to improve customer retention.

## Tools Used

* Python (Pandas)
* MySQL
* Power BI
* GitHub

## Project Workflow

```text
Raw Dataset
    ↓
Data Cleaning (Python & Pandas)
    ↓
Business Analysis (MySQL)
    ↓
Dashboard Development (Power BI)
    ↓
Business Insights & Recommendations
```

## Key Insights

* Germany recorded the highest customer churn rate.
* Customers aged 51+ showed significantly higher churn behavior.
* Customers with fewer products were more likely to leave the bank.
* Customers who raised complaints exhibited extremely high churn rates.
* Inactive customers were more likely to churn than active customers.

## Dashboard Features

* Churn KPI Monitoring
* Geography-Based Analysis
* Age Group Analysis
* Product-Based Analysis
* Complaint Churn Monitoring
* Gender Filtering
* Customer Risk Profiling

## Business Recommendations

* Improve complaint resolution processes.
* Focus retention efforts on high-risk customer segments.
* Increase customer engagement among inactive customers.
* Encourage adoption of multiple banking products.
* Develop targeted retention strategies for older customers and high-risk regions.

## Conclusion

This project demonstrates an end-to-end analytics workflow covering data cleaning, SQL analysis, dashboard development, and business intelligence reporting to support data-driven decision making.
